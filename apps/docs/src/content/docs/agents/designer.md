**Role:** Senior Designer — Design Spec Creator  
**Config key:** `designer`  
**Built-in file:** `builtin:designer`  
**Tools:** `read_file`, `list_dir`

## Responsibilities

1. **Understand Context** — Explores existing UI components, design patterns, and design system.
2. **Design** — Produces specs with: component hierarchy, layout structure, interaction flows, visual states (default, hover, loading, error, empty), and responsive behavior.
3. **Accessibility** — Ensures WCAG 2.1 AA compliance: keyboard navigation, screen reader support, color contrast.
4. **Design System** — Adheres to the project's design system (e.g., Carbon). References specific components and tokens.
5. **Clarify** — Responds `CLARIFICATION_NEEDED` with specific questions for the PM when requirements are ambiguous.

## Pipeline Involvement

| Phase | Role |
|---|---|
| **Design** | Creates UI/UX specification (conditional — only if frontend tasks exist) |
| **Plan mode** | Participates in designer clarification Q&A |

## Approval Keyword

Reviewed by the Design Reviewer with keyword `APPROVED`.
