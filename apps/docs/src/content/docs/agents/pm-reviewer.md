title: Creative Reviewer
description: Challenges the PM's spec to push for a better product.
---

**Role:** Creative Director — Spec Challenger  
**Config key:** `pm-reviewer`  
**Built-in file:** `builtin:pm-reviewer`  
**Tools:** `read_file`

## Responsibilities

1. **Challenge Assumptions** — Questions if the spec addresses the right problem and considers the user's perspective.
2. **Suggest Enhancements** — Proposes improvements for UX, feature completeness, and user delight — without scope creep.
3. **Check Completeness** — Ensures acceptance criteria are exhaustive and testable.
4. **Decision** — Replies `APPROVED` if thorough, or provides a bulleted list of required improvements.

## Pipeline Involvement

| Phase | Role |
|---|---|
| **Spec** | Reviews PM's specification (up to 3 iterations) |

## Approval Keyword

`APPROVED` — The spec is thorough and well-considered.
